🏥 Doctor Dashboard System - Testing Guide

📋 Overview

This guide will help you test the complete doctor dashboard system. The system includes:

  • ✅ Admin dashboard for managing doctors and assigning patients
  • ✅ Doctor dashboard with patient management and scheduling
  • ✅ Patient assignment system with ward numbers and priorities
  • ✅ Schedule management with time slots and activities
  • ✅ Role-based authentication and authorization

🚀 Quick Start

Step 1: Start the Backend Server

cd medi/medimeal-backend
npm install
npm start

Server should start on http://localhost:5000

Step 2: Start the Frontend Server

cd medi/medimeal-frontend
npm install
npm start

Frontend should start on http://localhost:3000

Step 3: Seed Test Data (Optional)

cd medi/medimeal-backend
node scripts/seed-test-data.js

This creates test users, patients, and assignments for testing.

🔧 API Testing

Run Automated Tests

cd medi/medimeal-backend
node scripts/test-doctor-system.js

This will run comprehensive API tests covering all endpoints.

Manual API Testing

Use Postman or curl to test endpoints:

# Test doctor dashboard curl -H "Authorization: Bearer YOUR_DOCTOR_TOKEN" \ http://localhost:5000/api/doctor/dashboard # Test patient assignments curl -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \ http://localhost:5000/api/admin/patient-assignments

🐛 Troubleshooting

Common Issues

  • Database Connection: Ensure MongoDB is running
  • Port Conflicts: Check if ports 3000 and 5000 are available
  • Authentication: Verify JWT_SECRET is set in .env
  • CORS Issues: Check CORS configuration in server.js

Check Server Logs

Monitor backend server logs for errors:

# Backend logs cd medi/medimeal-backend npm start # Frontend logs cd medi/medimeal-frontend npm start

✅ Success Criteria

The system is working correctly if:

  • ✅ Admin can create doctor accounts
  • ✅ Admin can assign patients to doctors
  • ✅ Doctors can log in and see their dashboard
  • ✅ Dashboard displays all required elements from the image
  • ✅ Patient list shows assigned patients with correct details
  • ✅ Calendar and schedule sections are functional
  • ✅ All API endpoints return correct data
  • ✅ Role-based access control works properly
